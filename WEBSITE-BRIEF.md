# Bots & Bones Website Build Prompt

You are building the official Bots & Bones website.

This is NOT a generic AI agency website.

The goal is to create a premium operational universe that combines:
- AI systems,
- structured workflows,
- generative creative,
- collectible product aesthetics,
- and cinematic industrial design.

The website should feel like:
- a modular operating system,
- a collectible product ecosystem,
- and a creative technology studio at the same time.

Avoid anything that feels:
- corporate,
- startup-generic,
- SaaS-template-like,
- crypto/web3,
- neon cyberpunk,
- or productivity-dashboard-heavy.

The visual language should feel intentional, premium, calm and engineered.

---

# CORE CONCEPT

The entire brand is built around three layers:

## Bots
Intelligence layer.
Agents, automation, memory, orchestration and execution.

## Bones
Structure layer.
Pipelines, workflows, databases, operational systems and logic.

## Skin
Creative layer.
Generative visuals, interfaces, content and synthetic media.

These layers should subtly appear across the website through:
- tags,
- labels,
- iconography,
- layouts,
- and motion.

Do NOT over-explain them.

---

# VISUAL DIRECTION

The aesthetic combines:
- Teenage Engineering
- Apple hardware presentations
- premium collectible packaging
- Japanese gadget packaging
- industrial systems
- cinematic product renders
- modular electronics
- operational interfaces

Use:
- deep blacks,
- warm off-whites,
- subtle metallics,
- translucent plastics,
- soft shadows,
- restrained glow,
- cinematic lighting,
- negative space.

The website must feel premium and calm.

---

# MASCOT SYSTEM

The mascot is a core part of the identity.

It is NOT a decorative illustration.

It acts as:
- a system metaphor,
- a collectible entity,
- a visual interface for capabilities,
- and an operational character.

The mascot system is modular and NFT-inspired:
- different variants,
- different modules,
- different operational states,
- different packaging formats.

Use the provided mascot assets across the site.

---

# HERO SECTION

The hero should feature:
- one large mascot render,
- preferably an exploded-view or assembly animation,
- minimal UI clutter,
- cinematic presentation.

Do NOT place excessive labels around the mascot.

The mascot should breathe visually.

The hero should feel:
- iconic,
- premium,
- engineered,
- slightly mysterious.

Use one of the provided headline systems.

---

# TAG SYSTEM

Tags are extremely important.

Tags should work as:
- UI language,
- navigation language,
- system vocabulary,
- and storytelling devices.

Tags should appear:
- in sections,
- inside cards,
- in hover states,
- near capabilities,
- and inside cases.

Use restrained microtypography.

Avoid visual noise.

---

# MICROTYPOGRAPHY

Microtypography is part of the identity.

Use the provided microtype/iconography system carefully and sparingly.

Microtype should feel:
- operational,
- subtle,
- technical,
- intentional.

Never overload screens with labels.

Microtypography should enhance realism, not become decoration spam.

Use it similarly to:
- industrial products,
- electronic devices,
- engineering labels,
- collectible packaging.

---

# SERVICES / CAPABILITIES SECTION

This section should NOT feel like a services grid.

Instead, present capabilities as:
- collectible operational modules,
- premium packaged units,
- Barbie-style blister packs,
- modular AI products.

Each card should contain:
- mascot variant,
- accessories/modules,
- tags,
- operational labels,
- product-like naming.

The section should feel:
- playful,
- premium,
- engineered,
- collectible.

---

# FLAGSHIP SYSTEMS

The website contains three flagship systems:
- ContentOS
- LogisticsOS
- AgentOS

These are NOT SaaS dashboards.

They are system archetypes and operational patterns.

Each system should have:
- its own mascot variant,
- its own visual atmosphere,
- distinct tags,
- unique operational mood.

Use cinematic system visuals instead of generic dashboards.

---

# CASES

Cases should NOT use mascots.

Cases should use:
- real visuals,
- interfaces,
- content,
- motion frames,
- product imagery,
- operational graphics.

Cases should demonstrate combinations of:
- Bots,
- Bones,
- Skin.

Use tags to communicate this.

---

# GENERATE YOUR UNIT

The website includes an interactive section where users can generate their own Bots & Bones unit.

This mechanic should feel:
- premium,
- collectible,
- playful,
- and highly visual.

Users should be able to:
- generate multiple mascot variants,
- explore modules/accessories,
- unlock more generations after leaving contact information.

This should feel like:
- a collectible generator,
- not a lead form.

---

# MOTION

Motion should be:
- slow,
- cinematic,
- physical,
- subtle,
- premium.

Preferred motion styles:
- floating assembly,
- exploded-view transitions,
- soft idle movement,
- modular rearrangement,
- hover physics,
- operational transitions.

Avoid:
- flashy animations,
- aggressive parallax,
- overactive motion,
- “AI startup” effects.

---

# FINAL FEELING

The website should feel like:
- an operational design universe,
- a modular creative system,
- and a premium collectible technology brand.

The result should be memorable, tactile and emotionally designed.

The user should leave with the feeling that:
“These people are building systems, not just content.”
