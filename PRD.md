# PRD — Bots & Bones Mascot Generator

Продуктовая спека инструмента. Отвечает на вопросы: **что строим, зачем, для кого,
что в скоупе, что НЕ в скоупе, что считаем «готово»**.

Last updated: 2026-05-29.

---

## 1. Проблема и цель

Нужен инструмент, который превращает грубую идею персонажа в **бренд-безопасный**
маскот-юнит и генерит его изображение в едином визуальном языке Bots & Bones.

Цель: быстро получать консистентных маскотов (коллекционный продуктовый стиль),
не уезжая в «генерик AI-эстетику» и не нарушая палитру/правила бренда.

## 2. Для кого

- Команда студии: дизайн, контент, продукт.
- Внутренний инструмент (local-first), не публичный сервис на этом этапе.

## 3. Что строим (скоуп)

- Локальный веб-UI для сборки промптов маскота.
- Генерация изображений через FAL `fal-ai/nano-banana-2/edit`.
- Расширение идеи через OpenAI (`OPENAI_MODEL`, сейчас `gpt-5-mini`).
- Два режима:
  - **Auto brief** — короткая идея → концепт → ровно 9 редактируемых частей.
  - **Manual parts** — ручное редактирование 9 компонентных промптов.
- Редактор промпта (rebuild / copy / ручная правка).
- Валидация цветов бренда на фронте и бэке.
- Локальная галерея в `_generated/gallery.json`.
- Переиспользование результата как нового источника.
- Info-страница: объяснение пайплайна, примеры, система частей, чек-листы.

## 4. Что НЕ в скоупе (сейчас)

- Публичный хостинг / мультиюзер / авторизация.
- Лид-захват и «free generations» с сайта (это про маркетинговый сайт — см.
  [WEBSITE-BRIEF.md](WEBSITE-BRIEF.md)).
- Видео-рендер — реализован (Remotion, self-hosted): картинка + анимированный заголовок
  → `.mp4`. Спека: [REMOTION-PLAN.md](REMOTION-PLAN.md). Деплой: [DEPLOY.md](DEPLOY.md).

## 5. Пайплайн (как работает продукт)

1. Выбрать исходного маскота (source).
2. Ввести короткую идею (Auto brief) или переключиться в Manual parts.
3. OpenAI делает концепт-пасс: title, archetype, story, visual principle,
   signature details, avoid rules.
4. OpenAI раскладывает концепт ровно в **9 физических частей**:
   - Display / Face
   - Head Attachments
   - Body Shell / Material
   - Front Controls / Buttons
   - Storage Slots / Cavities
   - Arms / Hands
   - Modules / Accessories
   - Stickers / Labels / Markings
   - Wear / Finish / Status Lights
5. Приложение собирает финальный промпт: preservation rules + brand color rules +
   concept + parts + modifiers + context + restrictions.
6. Фронт и бэк валидируют промпт.
7. FAL Nano Banana 2 редактирует исходное изображение.
8. Сервер скачивает результат в `_generated/` и индексирует в `gallery.json`.

## 6. Бренд-гардрейлы (обязательны)

Канонический источник цвета — [DESIGN.md](DESIGN.md).

**Разрешённые hex:** `#f4f2ec` `#ebe8df` `#111111` `#0b0b0b` `#e8e5dc` `#dedacf`
`#171717` `#c8c4ba` `#ff3b30` `#49f7a5`.

**Запрещённые слова-дрейф цвета:** `cyan` `magenta` `yellow` `colorful` `rainbow`
`purple` `orange`.

Различия юнитов должны идти от материала, модулей, лейблов, маркировок, статус-лампочек,
прозрачности, износа и аксессуаров — **не** от случайных колорвеев.

## 7. Правила исходных артефактов

Source-изображение — референс для: позы, камеры, света, качества материала, стиля
рендера, пропорций маскота. **Это не чек-лист аксессуаров для сохранения.**

Шаблонные артефакты редактируемы и по умолчанию не должны выживать: очки/линзы/оправы,
декоративные цветы, галстуки/воротники/рубашки/клетчатая ткань, рюкзаки, novelty-марки,
снек-паки/телефоны/сигаретные пачки, сторонние логотипы, прочие случайные аксессуары.

Если концепт их не требует — промпт удаляет их, нейтрализует в бренд-safe поверхность
или конвертирует в релевантный модуль.

## 8. Критерии «готово» (для фич этого инструмента)

- Промпт проходит валидацию палитры (фронт + бэк) без запрещённых цветов/hex.
- Auto brief стабильно отдаёт ровно 9 частей.
- Результат сохраняется в `_generated/` и появляется в галерее.
- Сгенерированный юнит читается как Bots & Bones, а не как генерик-бот.

## 9. Технический справочник

**Routes**
- `/` — Generator, плотный 3-колоночный воркбенч.
- `/#generator-v2` — Steps V2, линейный сверху-вниз layout (та же логика/состояние).
- `/#info` — пайплайн, примеры, диаграммы, live-checks.
- `/#flow` — legacy-алиас на Info.
- `/#gallery` — галерея.

**Валидация**
```bash
node --check server.js
node --check public/app.js
rg -n "\b(cyan|magenta|yellow|colorful|rainbow|purple|orange)\b" _unit-gen public server.js
```
Бэкенд дополнительно проверяет неизвестные hex против разрешённой палитры.

**Файлы вывода:** `_generated/` (изображения), `_generated/gallery.json` (индекс).
